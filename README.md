# ADP Community

👋 Welcome to **The AI Docs Project (ADP) Community**! We're thrilled you're interested in participating and contributing.

At ADP, we value inclusivity, collaboration, and respect. We welcome contributions, questions, and ideas from anyone who shares our mission of building accessible and high-quality technical documentation.

This repository serves as the **central hub for community discussions, feedback, and collaboration** across ADP projects.


## Getting Started

1. Browse existing discussions to see if your topic has already been addressed.  
2. Start a new discussion in the appropriate category.  
3. Participate by commenting, sharing ideas, and providing feedback.
   

## GitHub Discussions

We encourage our community to start and engage in discussions about topics related to ADP projects. Each discussion should focus on a single topic to keep conversations clear and actionable.

**Note:** This repo is for **discussion and feedback only**.  
- If you want to report a bug or request a feature in a specific ADP repo, please open an issue in that repository.  
- Use discussions for sharing ideas, brainstorming solutions, or asking questions about ADP projects.



## Discussion Categories

| Category | Purpose |
|----------|---------|
| 📣 Announcements | ADP staff share important updates, release notes, and community news. |
| ✏️ Documentation | Feedback or ideas about content, guides, templates, and style guides. |
| 🛠️ Tooling & Code | Discussion about ADP tooling, scripts, automation, and integration. |
| 🎨 UX & Design | Suggestions for improving documentation layout, design, and readability. |
| 🌐 Localization | Questions or updates about translations and internationalization of ADP content. |
| 🤖 Community Feedback | General feedback, feature requests, or questions that don't fit other categories. |
| 📊 Polls | Reserved for ADP staff to collect input from the community. |

> **Tip:** Before starting a discussion, choose the category that best fits your topic. This keeps the forum organized and helps others find relevant conversations.



## Participation Guidelines

To maintain a welcoming and productive community, please follow these principles:

### Expected behaviours
- **Be respectful:** Value different viewpoints and ideas. Disagreements are fine, but treat others with courtesy.  
- **Communicate clearly and professionally:** Be honest but polite in your messages.  
- **Be inclusive:** Encourage diverse perspectives; invite everyone to participate.  
- **Listen and learn:** Be open to ideas that may improve your own thinking or work.  
- **Lead by example:** Contribute thoughtfully, hold yourself accountable, and help maintain a positive environment.

### Unacceptable behaviours
- **Violence or threats** – Online or offline threats are not tolerated.  
- **Personal attacks** – Insults, harassment, or belittling others is unacceptable.  
- **Derogatory language** – Avoid discriminatory, offensive, or biased language.  
- **Unwelcome sexual attention** – Includes inappropriate comments, jokes, or advances.  
- **Disruptive behaviour** – Sustained interference in discussions or events will not be tolerated.  

By participating, you agree to **follow these guidelines** and help create a safe, inclusive, and productive space for everyone.


Together, we can build a strong, collaborative, and inclusive ADP community! 🎉
